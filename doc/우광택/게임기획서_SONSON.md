# SONSON 웹버전 게임 기획서

## 1. 게임 개요

### 게임명

**SONSON 웹버전**

### 소속 프로젝트

**추억의 오락실** 팀 프로젝트 내 개별 게임

### 게임 장르

횡스크롤 플랫폼 슈팅 게임

### 개발 목적

본 게임은 추억의 아케이드 게임 **SONSON**의 핵심 플레이 감각을 웹 브라우저에서 재현하는 것을 목표로 한다.

플레이어는 자동으로 스크롤되는 화면 속에서 여러 층의 발판을 오르내리며 적을 피하고, 공격하며, 아이템을 획득해 점수를 얻는다. 팀 프로젝트 전체 구조 안에서는 하나의 독립 게임으로 동작하되, CREDIT 시스템은 공통 모듈을 사용한다.

---

## 2. 핵심 콘셉트

> 자동으로 앞으로 나아가는 횡스크롤 화면에서, 플레이어가 층을 오르내리며 적을 피하고 공격하는 레트로 아케이드 액션 슈팅 게임

SONSON 웹버전의 핵심 재미는 다음 세 가지이다.

1. **자동 스크롤의 긴장감**

   * 화면은 계속 앞으로 진행된다.
   * 플레이어는 멈춰 있을 수 없고, 계속 변화하는 상황에 대응해야 한다.

2. **여러 층 발판 이동**

   * 화면에는 여러 개의 가로 발판이 존재한다.
   * 플레이어는 위아래로 층을 이동하며 적을 피하거나 아이템을 획득한다.

3. **단순하지만 빠른 공격**

   * 플레이어는 전방으로 탄을 발사해 적을 제거한다.
   * 조작은 단순하지만 적 배치와 스크롤 때문에 순발력이 필요하다.

---

## 3. 팀 프로젝트 내 역할

SONSON 웹버전은 전체 프로젝트 **추억의 오락실**에 포함되는 4개 게임 중 하나이다.

전체 프로젝트에서 공통으로 사용하는 데이터는 CREDIT뿐이다.
SONSON 게임은 다른 게임과 점수, 스테이지, 캐릭터, 아이템 정보를 공유하지 않는다.

단, 다음 공통 규칙은 반드시 따른다.

* 게임 시작 시 CREDIT 100원을 차감한다.
* 게임 오버 후 다시 시작할 때도 CREDIT 100원을 차감한다.
* CREDIT이 100원 미만이면 게임을 시작할 수 없다.
* CREDIT 값은 직접 수정하지 않고 공통 CREDIT 모듈을 사용한다.
* 게임 화면에는 현재 CREDIT을 표시한다.
* 로비로 돌아가기 버튼을 제공한다.

## 4. 사용자 흐름

### 4.1 로비에서 게임 입장

1. 사용자가 프로젝트 메인 화면에 접속한다.
2. CREDIT을 확인한다.
3. 게임 목록에서 **SONSON**을 선택한다.
4. SONSON 게임 화면으로 이동한다.
5. 이 시점에서는 CREDIT이 차감되지 않는다.

### 4.2 게임 시작

1. 사용자가 SONSON 화면에서 **게임 시작** 버튼을 누른다.
2. `spendCredit()` 함수를 호출한다.
3. CREDIT이 100원 이상이면 100원이 차감된다.
4. 게임이 시작된다.
5. CREDIT이 부족하면 게임은 시작되지 않는다.

### 4.3 게임 플레이

1. 플레이어는 자동 스크롤되는 화면에서 캐릭터를 조작한다.
2. 위아래 층을 이동하며 적을 피한다.
3. 공격 버튼으로 전방에 탄을 발사한다.
4. 적을 제거하면 점수가 증가한다.
5. 과일 아이템을 획득하면 추가 점수를 얻는다.
6. 적이나 적의 공격에 닿으면 생명이 감소한다.
7. 생명이 모두 사라지면 게임 오버가 된다.

### 4.4 게임 오버 후 재시작

1. 게임 오버 화면이 표시된다.
2. 사용자는 **다시 시작** 버튼을 누를 수 있다.
3. 다시 시작 시에도 `spendCredit()`을 호출한다.
4. CREDIT이 충분하면 100원이 차감되고 새 게임이 시작된다.
5. CREDIT이 부족하면 로비에서 충전하라는 안내를 표시한다.

## 6. 조작 방식

### PC 키보드 기준

| 조작              | 키     |
| --------------- | ----- |
| 왼쪽 이동 / 속도 늦추기  | ←     |
| 오른쪽 이동 / 속도 높이기 | →     |
| 위층으로 이동         | ↑     |
| 아래층으로 이동        | ↓     |
| 공격              | Space |
| 일시정지            | P     |
| 로비로 돌아가기        | 버튼 클릭 |

### 조작 설계 방향

원작 감성을 살리기 위해 플레이어는 자유롭게 점프하는 방식보다는, 정해진 층 사이를 위아래로 이동하는 방식으로 구현한다.

좌우 방향키는 캐릭터의 위치 조정 및 상대적 이동 속도 조절에 사용한다.
화면은 기본적으로 자동으로 오른쪽으로 스크롤된다.

## 7. 핵심 게임 규칙

### 7.1 자동 스크롤

* 게임이 시작되면 화면이 자동으로 오른쪽 방향으로 스크롤된다.
* 플레이어는 화면 안에서 상대적으로 움직인다.
* 플레이어가 화면 왼쪽 끝으로 밀려나면 위험 상태가 된다.
* 화면 밖으로 밀려나거나 적에게 맞으면 생명이 감소한다.

### 7.2 발판 구조

* 화면에는 여러 개의 가로 발판이 존재한다.
* 기본 구현에서는 6개 층을 사용한다.
* 플레이어는 위/아래 키를 눌러 인접한 발판으로 이동한다.
* 각 발판에는 적, 아이템, 장애물이 등장할 수 있다.

### 7.3 공격

* 플레이어는 Space 키로 전방에 탄을 발사한다.
* 탄은 오른쪽 방향으로 이동한다.
* 탄이 적에게 닿으면 적이 사라지고 점수가 증가한다.
* 한 번에 화면에 존재할 수 있는 플레이어 탄 수를 제한하여 난이도를 유지한다.

### 7.4 적

기본 MVP에서는 다음 적을 구현한다.

| 적 종류  | 특징                |
| ----- | ----------------- |
| 기본 적  | 일정 속도로 왼쪽 방향 이동   |
| 점프 적  | 발판 사이를 이동하거나 튀어오름 |
| 투사체 적 | 일정 간격으로 공격 발사     |

추후 확장 시 박쥐형 적, 폭탄형 적, 빠른 돌진형 적 등을 추가할 수 있다.

### 7.5 아이템

기본 아이템은 과일로 설정한다.

| 아이템   | 효과                               |
| ----- | -------------------------------- |
| 작은 과일 | 점수 +100                          |
| 큰 과일  | 점수 +500                          |
| 특수 과일 | 화면 내 적을 점수 아이템으로 전환 또는 보너스 점수 지급 |

MVP에서는 작은 과일과 큰 과일만 먼저 구현한다.
특수 과일은 확장 기능으로 둔다.

### 7.6 생명과 게임 오버

* 플레이어는 기본 생명 3개로 시작한다.
* 적과 충돌하거나 적의 공격에 맞으면 생명 1개가 감소한다.
* 피격 후 짧은 무적 시간이 주어진다.
* 생명이 0이 되면 게임 오버가 된다.

점수는 게임 플레이의 성취감을 주기 위한 기본 요소이다.

| 행동       |    점수 |
| -------- | ----: |
| 기본 적 제거  |  +100 |

MVP에서는 온라인 랭킹을 구현하지 않는다.
점수는 현재 플레이 세션 안에서만 표시한다.

## 9. 스테이지 구성

### MVP 스테이지

MVP에서는 1개의 짧은 스테이지만 구현한다.

구성은 다음과 같다.

1. 시작 구간

   * 기본 적이 천천히 등장한다.
   * 조작을 익히는 구간이다.

2. 중간 구간

   * 적 수가 증가한다.
   * 위아래 층 이동이 필요해진다.
   * 과일 아이템이 등장한다.

3. 후반 구간

   * 빠른 적과 투사체 적이 등장한다.
   * 화면 스크롤 속도가 약간 빨라진다.

4. 클리어 또는 게임 오버

   * 일정 시간 생존하면 스테이지 클리어
   * 생명이 0이 되면 게임 오버

### 확장 스테이지

추후 시간이 허락하면 다음을 추가할 수 있다.

* 스테이지 2
* 보스 적
* 보너스 아이템
* 난이도 상승
* 배경 변화
* 2인 협동 모드

## 12. 공통 CREDIT 연동 요구사항

SONSON 웹버전은 팀 프로젝트의 공통 CREDIT 시스템을 반드시 사용한다.

### 필수 함수

```js
getCredit()
spendCredit()
updateCreditDisplay()
```

### 게임 시작 흐름

```js
function handleStartButtonClick() {
  if (spendCredit()) {
    updateCreditDisplay();
    startGame();
  } else {
    alert("CREDIT이 부족합니다. 로비에서 충전해 주세요.");
  }
}
```

### 다시 시작 흐름

```js
function handleRestartButtonClick() {
  if (spendCredit()) {
    updateCreditDisplay();
    restartGame();
  } else {
    alert("CREDIT이 부족합니다. 로비에서 충전해 주세요.");
  }
}
```

### 금지 사항

* 게임 내부에서 CREDIT 값을 직접 수정하지 않는다.
* 별도의 CREDIT 변수를 만들지 않는다.
* `localStorage.setItem("credit", ...)`을 게임 코드에서 직접 호출하지 않는다.
* 게임 입장 시 CREDIT을 차감하지 않는다.
* 게임 시작 전에 CREDIT이 차감되지 않도록 한다.

## 18. 완료 기준

SONSON 웹버전은 다음 조건을 만족하면 MVP 완료로 본다.

1. 팀 프로젝트 로비에서 SONSON 게임으로 이동할 수 있다.
2. SONSON 화면에서 현재 CREDIT을 확인할 수 있다.
3. 게임 시작 시 CREDIT 100원이 차감된다.
4. CREDIT이 부족하면 게임이 시작되지 않는다.
5. 플레이어가 자동 스크롤 화면에서 이동하고 공격할 수 있다.
6. 적이 등장하고 충돌 판정이 작동한다.
7. 점수와 생명이 표시된다.
8. 생명이 0이 되면 게임 오버가 된다.
9. 다시 시작 시 CREDIT 100원이 차감된다.
10. 로비로 돌아갈 수 있다.
