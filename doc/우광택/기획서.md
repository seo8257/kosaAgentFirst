# 팀 프로젝트 기획서

## 프로젝트명: 추억의 노래방

## 1. 프로젝트 개요

**추억의 노래방**은 4명의 팀원이 각자 하나씩 만든 추억의 오락실 스타일 웹게임을 하나의 첫 화면에서 선택해 즐길 수 있는 웹 기반 미니게임 플랫폼이다.

사용자는 첫 화면에서 가상의 CREDIT을 충전하고, 원하는 게임을 선택해 입장할 수 있다. 각 게임은 독립적으로 기획·개발되지만, 모든 게임은 공통 CREDIT 시스템을 사용한다.

게임을 새로 시작하거나 다시 시작할 때마다 CREDIT 100원이 차감된다.

본 프로젝트는 상업적 배포 목적이 아닌 팀 프로젝트 및 학습용 웹게임 제작을 목표로 한다.

---

## 2. 프로젝트 목적

이 프로젝트의 목적은 다음과 같다.

1. 팀원 각자가 독립적인 웹게임을 기획하고 개발한다.
2. 여러 개의 개별 게임을 하나의 웹 서비스로 통합한다.
3. 공통 CREDIT 시스템을 통해 게임 간 최소한의 데이터 공유 구조를 설계한다.
4. HTML, CSS, JavaScript 기반의 웹게임 제작 경험을 쌓는다.
5. Codex 또는 AI 코딩 도구를 활용하여 기획, 구현, 디버깅, 리팩터링 과정을 경험한다.

---

## 3. 핵심 콘셉트

> 노래방에서 원하는 노래를 고르듯, 사용자가 원하는 추억의 오락실 게임을 선택해 플레이하는 웹 기반 레트로 게임 플랫폼

서비스명인 **추억의 노래방**은 실제 노래방 서비스가 아니라, “선택하고 즐긴다”는 노래방의 경험을 게임 선택 화면에 비유한 이름이다.

사용자는 첫 화면에서 CREDIT을 충전하고, 4개의 게임 중 하나를 선택해 플레이한다.
각 게임은 독립적인 게임성을 가지지만, CREDIT 차감 방식은 모든 게임에서 동일하게 작동한다.

---

## 4. 주요 사용자 경험

사용자 흐름은 다음과 같다.

1. 사용자가 첫 화면에 접속한다.
2. 현재 CREDIT을 확인한다.
3. CREDIT이 부족하면 CREDIT을 충전한다.
4. 4개의 게임 중 하나를 선택한다.
5. 선택한 게임 화면으로 이동한다.
6. 게임 시작 버튼을 누른다.
7. CREDIT 100원이 차감된다.
8. 게임을 플레이한다.
9. 게임 오버 후 다시 시작할 수 있다.
10. 다시 시작할 때도 CREDIT 100원이 차감된다.
11. 사용자는 로비로 돌아가 다른 게임을 선택할 수 있다.

---

## 5. 전체 서비스 구조

서비스는 크게 두 부분으로 구성된다.

### 5.1 첫 화면 / 로비

첫 화면은 사용자가 CREDIT을 충전하고 게임을 선택하는 공간이다.

주요 기능은 다음과 같다.

* 서비스 제목 표시
* 현재 CREDIT 표시
* CREDIT 충전 버튼 제공
* 4개 게임 선택 카드 표시
* 각 게임으로 이동하는 버튼 제공

---

### 5.2 개별 게임 화면

각 게임 화면은 팀원별로 독립적으로 개발한다.

각 게임의 장르, 규칙, 화면 구성, 난이도, 그래픽 표현 방식은 각 팀원이 자유롭게 결정한다.
단, CREDIT 처리 방식은 반드시 공통 CREDIT 모듈을 사용해야 한다.

각 게임 화면에 공통적으로 포함되면 좋은 요소는 다음과 같다.

* 게임 제목
* 현재 CREDIT 표시
* 게임 시작 버튼
* 점수 표시
* 게임 오버 표시
* 다시 시작 버튼
* 로비로 돌아가기 버튼

## 6. 팀 구성 및 역할

본 프로젝트는 4인 1팀으로 진행한다.

| 구분    | 역할                                 |
| ----- | ---------------------------------- |
| 팀원 1  | 게임 1 기획 및 개발                       |
| 팀원 2  | 게임 2 기획 및 개발                       |
| 팀원 3  | 게임 3 기획 및 개발                       |
| 팀원 4  | 게임 4 기획 및 개발                       |
| 공통 협의 | 로비 화면, CREDIT 시스템, 파일 구조, 공통 규칙 정의 |

각 팀원은 자신의 게임을 독립적으로 개발하되, 팀에서 정의한 공통 CREDIT 처리 규칙을 반드시 따른다.

## 7. 게임 개발 범위

각 게임은 “추억의 오락실 슈팅 게임”을 기본 방향으로 한다.

다만 각 팀원이 서로 다른 게임성을 만들 수 있도록 세부 장르는 자유롭게 정한다.

예시 방향은 다음과 같다.

| 게임   | 예시 콘셉트           |
| ---- | ---------------- |
| 게임 1 | 우주 침략자 스타일 슈팅 게임 |
| 게임 2 | 세로 스크롤 비행 슈팅 게임  |
| 게임 3 | 탱크 슈팅 게임         |
| 게임 4 | 보스전 중심 탄막 슈팅 게임  |

각 게임은 원작 게임을 그대로 복제하기보다, 추억의 오락실 게임 감성을 참고하여 웹에서 작동하는 미니게임으로 구현한다.

## 8. 공통 CREDIT 시스템

본 프로젝트에서 모든 게임이 공유하는 유일한 데이터는 CREDIT이다.

CREDIT은 원화 단위로 표시한다.

예:

```text
CREDIT: 500원
```

### 8.1 CREDIT 충전

CREDIT 충전은 첫 화면에서만 가능하다.

예시 충전 버튼은 다음과 같다.

```text
[100원 충전] [500원 충전] [1000원 충전]
```

또는 오락실 감성을 살려 다음과 같이 표현할 수 있다.

```text
[동전 1개 넣기: 100원]
[동전 5개 넣기: 500원]
[지폐 넣기: 1000원]
```

---

### 8.2 CREDIT 차감

각 게임에서 새 게임을 시작할 때마다 CREDIT 100원이 차감된다.

차감 시점은 다음과 같다.

* 게임 화면에 입장할 때는 차감하지 않는다.
* 게임 시작 버튼을 눌렀을 때 100원을 차감한다.
* 게임 오버 후 다시 시작할 때도 100원을 차감한다.
* CREDIT이 100원 미만이면 게임을 시작할 수 없다.

---

### 8.3 CREDIT 부족 처리

CREDIT이 부족할 경우 게임을 시작하지 않고 안내 메시지를 표시한다.

예시:

```text
CREDIT이 부족합니다.
로비에서 CREDIT을 충전해 주세요.
```

---

## 9. 공통 개발 규칙

각 게임 개발자는 CREDIT 값을 직접 수정하지 않는다.

모든 CREDIT 관련 처리는 공통 CREDIT 모듈을 통해서만 수행한다.

공통 CREDIT 모듈에서 제공할 함수 예시는 다음과 같다.

```js
getCredit()
addCredit(amount)
canPlay()
spendCredit()
updateCreditDisplay()
```

각 게임에서 게임 시작 시에는 다음과 같은 흐름을 사용한다.

```js
if (spendCredit()) {
  startGame();
} else {
  alert("CREDIT이 부족합니다. 로비에서 충전해 주세요.");
}
```

## 12. 프로젝트의 장점

이 프로젝트는 팀 프로젝트로 적합한 구조를 가진다.

1. 각 팀원이 독립적으로 개발할 수 있다.
2. 공통 데이터가 CREDIT 하나뿐이라 통합 난이도가 낮다.
3. 게임 결과물이 시각적으로 바로 확인된다.
4. Codex와 같은 AI 코딩 도구를 활용하기 좋다.
5. 발표 시 서비스 구조를 설명하기 쉽다.
6. “100원 넣고 한 판”이라는 레트로 감성이 분명하다.

---

## 13. 최종 목표

최종 결과물은 다음 조건을 만족하는 웹 서비스이다.

1. 첫 화면에서 CREDIT을 충전할 수 있다.
2. 첫 화면에서 4개의 게임 중 하나를 선택할 수 있다.
3. 각 게임은 독립적으로 실행된다.
4. 각 게임 시작 시 CREDIT 100원이 차감된다.
5. CREDIT이 부족하면 게임을 시작할 수 없다.
6. 게임 종료 후 다시 시작할 수 있다.
7. 다시 시작할 때도 CREDIT 100원이 차감된다.
8. 모든 게임은 동일한 CREDIT 값을 공유한다.
9. 사용자는 언제든 로비로 돌아갈 수 있다.

---

## 14. 한 문장 요약

**추억의 노래방은 4개의 레트로 슈팅 웹게임을 하나의 로비에서 선택해 즐기고, 모든 게임이 공통 CREDIT 시스템을 공유하는 팀 프로젝트형 미니게임 플랫폼이다.**
